# AI 架构师面试备忘录 (BMAD-Speckit-SDD-Flow 项目实战)

> 💡 **核心思想**：你是“AI 软件工厂的厂长”与“规则制定者”，而非单纯“大模型的打字员”或“传统背八股文的码农”。利用本项目的架构复杂度和 AI 管控流，对传统面试进行降维打击。

---

## 一、 核心定位与求职策略

### 1. 避开卷王赛道
- **不要投**：死磕 LeetCode 手撕算法题、传统 CRUD 增删改查的底层后端研发岗（如纯 Java/Go 业务开发）。
- **必须投**：
  - AI 研发效能工程师 (DevEx Engineer)
  - LLM 应用架构师 / Agentic Workflow 工程师
  - AI 代码教练 (AI Code Coach) / AI 研发布道师
  - 内部大模型评测与平台中台研发

### 2. 重新定义“技术基础”与“算法”
- **传统算法的贬值**：在 AI 时代，手撕红黑树等传统底层数据结构的能力正在被大模型迅速贬值。
- **你的高阶算法**：你掌握的是**大模型控制流算法（LLM Control Flow Algorithms）**：
  - **带回退的有限状态机 (FSM)**：嵌套 SDD 的 5 层节点，基于 `commit_hash` 硬锁死。
  - **启发式搜索收敛算法**：Party-Mode 100 轮多角色对抗 + 连续 3 轮无 Gap 跳出机制。

## 二、 简历与作品展示策略（防范专利冲突）

在申请国家发明专利之前，必须严格防止核心技术（Secret Sauce）过度公开导致丧失“新颖性”。

1. **简历降维展示（只写 What 和 Impact）**：
   - **错误写法**：放上所有的核心 Prompt 和底层解析正则。
   - **正确写法**：“设计并实现了一种嵌套式 SDD 多智能体开发管控系统（专利准备中），通过引入一票否决(Veto)引擎与多角色对抗辩论，将 AI 代码生成的架构偏离率降低了 40%，实现全链路客观量化评审。”
2. **隐藏核心机密**：
   - 绝不公开：具体的审计提示词（`audit-prompts.md`）、`veto_core_logic` 详细判定树。
3. **高情商话术包装**：
   - 当面试官索要全部源码时：“这部分的核心状态机控制算法与收敛引擎，我目前正在委托代理机构申请国家发明专利，为了保护知识产权，暂时不方便展示完整源码。但我可以给您详细讲一下它的顶层抽象思路和架构流转……”（**此时高级感直接拉满**）

## 三、 面试高频问题与满分话术应答

### Q1：遇到传统底层算法题如何化解？
> **面试官**：“你能手写一个红黑树的插入调整逻辑吗？”
> 
> **你（诚恳且自信）**：“面试官您好，老实说，如果要我立刻在白板上无 bug 地手撕红黑树的变色和左旋右旋逻辑，我可能做不到，因为最近我的技术精力并没有放在底层数据结构的重写上。我知道红黑树是用来解决极端情况下的二叉树平衡问题的。
> 
> 但在实际工程中，相比于手写造轮子，**我目前的强项在于‘高复杂度的系统流转控制’与‘AI 行为的边界约束’。** 比如在我的个人项目里，为了防止大模型输出发散，我手写了一套状态机算法，引入了基于权重的 Veto 一票否决树，以及多智能体在 3 轮无 Gap 时的强行收敛机制。这种**工程上的逻辑推演和系统级 debug 能力**，是我最核心的底子。如果您愿意，我很乐意给您展示一下我是如何通过代码控制这套系统的。”

### Q2：面对系统中尚未完全落地的功能如何解释？
> **面试官**：“你的系统中提到‘微调数据集与优化建议自动生成’，这个具体是怎么跑通的？”
> 
> **你（全局架构视野）**：“目前系统的主体控制流（嵌套 SDD 和打分引擎）已经全部 Run 起来了，全链路审计数据也全部按规范落盘到了 `scores.jsonl` 中。
> 
> 至于后续的 SFT 训练集生成模块，我在架构上已经设计了明确的数据流向：它的底层逻辑是通过脚本读取 `scores.jsonl` 中的 `run_id` 和 `commit_hash`，调用 `git diff` 提取触发 Veto 拦截前后的两次代码快照（Bad Code vs Good Code），然后组装成 OpenAI/HuggingFace 兼容的微调格式。
> 
> 在我的个人版本中，因为没有足够的算力去微调一个 32B 大模型，所以我目前的研发重心放在了**高质量的数据闭环采集**上（即 parse-and-write-score）。在 AI 工程界，高质量、带量化标签的原始数据湖往往比单纯的数据转化脚本更有价值。只要有了干净的数据，写个 Python 脚本做 SFT 导出只是半天的工作量。”

## 四、 你的面试终极武器库：5 张 Mermaid 架构图

在面试（尤其是视频面试）时，找机会将《技术交底书》中的 Mermaid 渲染图（或事先画好的流程图）发送或展示给面试官，践行 **Show, Don't Tell** 原则。

1. **附图 1（系统五层架构）**：
   - **作用**：展示你极具全局观的工程视野，证明你能 Hold 住从“需求输入”到“代码合并”的完整生命周期管控。
2. **附图 2（嵌套 SDD 状态机）**：
   - **作用**：讲解你如何解决“AI 幻觉”。强调**“前置节点 commit_hash 锁死”**机制，体现你对 DevOps 不可篡改性的深刻理解。
3. **附图 3（全链路审计落盘时序图）**：
   - **作用**：展现你懂高并发、数据流、异常熔断（如 `SCORE_WRITE_INPUT_INVALID`），这不是写单文件脚本，而是成熟的事件驱动中枢设计。
4. **附图 4（多角色对抗辩论 100 轮收敛算法）**：
   - **作用**：这是最具杀伤力的核心！用它展示你对 **Agentic Workflow（多智能体编排）** 的深刻理解。重点讲解“批判审计员”角色和“连续 3 轮无新 Gap 才打破 while 循环”的算法哲学。
5. **附图 5（Veto 一票否决机制处理流程）**：
   - **作用**：展示你如何将大模型定性的 NLP 诊断结果，转化为 DevOps 中定量的、冷酷的拦截降级动作（从 D 级打分直接转化为 Git 强制回退）。
