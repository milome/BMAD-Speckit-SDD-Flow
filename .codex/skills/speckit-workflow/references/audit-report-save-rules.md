# 审计报告保存规则（防死循环）

本文件定义审计子代理在保存报告时的行为约束，用于**防止「正在写入完整审计报告」重复输出导致的死循环**。

---

## 1. 问题现象

当 prompt 要求「将完整报告保存至 {报告路径}」时，部分模型会重复输出「正在写入完整审计报告」数百次，导致：
- 输出阻塞、无法完成
- 主 Agent 无法收到完整报告
- 审计流程无法收敛

## 2. 根因

- 模型将「保存」理解为需输出状态信息
- 「完整」可能被误解为需对报告的每个部分分别确认
- 缺乏明确停止信号，导致重复生成同一短语

## 3. 强制约束

**禁止**：保存报告时不得重复输出以下或类似状态信息：
- 「正在写入完整审计报告」
- 「正在保存」
- 「Writing complete audit report」

**必须**：
1. 使用 `write` 或 `search_replace` 工具**一次性**将完整报告内容写入指定路径
2. 完成后在报告结论中注明保存路径
3. 不输出任何重复性状态短语

## 4. 报告保存模板（供主 Agent 复制到 prompt）

当在审计 prompt 中包含「## 报告保存」时，使用以下模板：

```markdown
## 报告保存

**每轮报告（无论通过与否）均须保存**。将完整报告保存至：{报告路径}

**禁止**：不得重复输出「正在写入完整审计报告」「正在保存」等状态信息；使用 write 工具一次性写入完整内容即可，完成后在结论中注明保存路径。
```

## 5. 引用

在构建 PRD、TASKS、Story 等文档审计 prompt 时，若包含「报告保存」段落，须同时包含上述禁止条款。
