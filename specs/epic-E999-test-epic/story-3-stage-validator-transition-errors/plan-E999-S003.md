## 1. 实现目标

以最小变更完成 typed transition validation 能力落地，并确保 StoryStateManager 在推进阶段时统一复用该能力。

## 2. 实施步骤

### Phase 1：测试先行（RED）

1. 在 `tests/story-workflow/StageValidator.test.ts` 增加 `validateTransition` 的成功/失败断言。
2. 在 `tests/story-workflow/StoryStateManager.test.ts` 增加：
   - 共享 validator 的 skip/reverse 错误断言
   - `advanceStage()` 经过 validator 的断言
3. 在 `tests/story-workflow/integration.test.ts` 增加结构化错误验证。
4. 运行目标测试，确认 RED 失败，证明新增能力尚未实现。

### Phase 2：最小实现（GREEN）

1. 在 `StageValidator.ts` 中新增 `validateTransition`。
2. 将 `validateEntry` 改为复用 `validateTransition`。
3. 在 `StoryStateManager.ts` 中新增 `getCompletedStages()`。
4. 在 `advanceStage()` 中接入 `validateTransition`。

### Phase 3：回归验证

1. 重新运行三份 story-workflow 测试。
2. 确认 40 个测试全部通过。
3. 更新 tasks 与 state 语义。
