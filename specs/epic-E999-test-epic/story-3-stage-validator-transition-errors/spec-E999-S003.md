# Spec E999-S003：Stage Validator Transition Errors

**Epic**：E999 test-epic
**Story**：S003 stage-validator-transition-errors

---

## 1. 概述

本 story 以最小真实实现方式，为 `src/story-workflow` 引入可编程消费的阶段转换校验能力，替代仅返回字符串错误的模式。

目标范围：

1. `StageValidator` 新增 typed `validateTransition` 能力。
2. 非法转换必须返回结构化 `StageTransitionError`。
3. 至少覆盖三类非法转换：
   - prerequisite not met
   - skip not allowed
   - reverse not allowed
4. `StoryStateManager.advanceStage()` 统一经过该 typed 校验。
5. 相关单元测试与集成测试真实通过。

非目标：
- 不扩展 story-workflow 之外的模块。
- 不改动现有阶段模型与公共 API 范围之外的大量文件。

## 3. 功能规格

### 3.1 StageValidator typed transition validation

新增方法签名：

- `validateTransition(context: ValidationContext): Result<ValidationContext, StageTransitionError>`

#### 3.1.1 prerequisite not met

当目标阶段存在 prerequisite，且 `completedStages` 不包含该 prerequisite 时：

- 返回 `err(StageTransitionError)`
- `code = ErrorCodes.PREREQUISITE_NOT_MET`
- `fromStage = currentStage`
- `toStage = targetStage`

#### 3.1.2 skip not allowed

当 `targetStage` 不是 `currentStage` 的顺序下一个阶段，且目标位于更后阶段时：

- 返回 `err(StageTransitionError)`
- `code = ErrorCodes.SKIP_NOT_ALLOWED`

#### 3.1.3 reverse not allowed

当 `targetStage` 位于 `currentStage` 之前时：

- 返回 `err(StageTransitionError)`
- `code = ErrorCodes.REVERSE_NOT_ALLOWED`

#### 3.1.4 与 validateEntry 的兼容

`validateEntry` 继续保留原字符串错误返回格式，但内部应复用 `validateTransition`，避免双重逻辑漂移。

### 3.2 StoryStateManager 集成

`advanceStage()` 在完成当前阶段且确定 next stage 后，必须：

1. 基于当前 state 计算 `completedStages`
2. 调用 `validator.validateTransition(...)`
3. 若失败，直接返回 typed `StageTransitionError`
4. 若成功，才继续写入 stage history、切换 currentStage、记录 audit log
