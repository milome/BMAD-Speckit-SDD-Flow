### 核心模块

| 模块 | 职责 | 文件 |
|------|------|------|
| `parser` | 解析邮箱地址为 local-part 和 domain | `src/parser.ts` |
| `validator` | 执行 RFC 5322 正则验证 | `src/validator.ts` |
| `errors` | 错误代码和消息管理 | `src/errors.ts` |
| `index` | 主入口和类型导出 | `src/index.ts` |

## 数据模型

### EmailValidationResult

```typescript
interface EmailValidationResult {
  valid: boolean;
  error?: string;
  errorCode?: ValidationErrorCode;
}

type ValidationErrorCode =
  | 'EMPTY'
  | 'INVALID_FORMAT'
  | 'TOO_LONG';
```

### ParsedEmail

```typescript
interface ParsedEmail {
  localPart: string;
  domain: string;
  original: string;
}
```

## 接口契约

### 主函数

```typescript
/**
 * 验证邮箱地址格式
 * @param email - 待验证的邮箱地址
 * @returns EmailValidationResult - 验证结果
 */
function validateEmail(email: string): EmailValidationResult;
```

### 辅助函数

```typescript
/**
 * 解析邮箱地址
 * @param email - 邮箱地址
 * @returns ParsedEmail | null - 解析结果
 */
function parseEmail(email: string): ParsedEmail | null;

/**
 * 验证本地部分格式
 * @param localPart - @ 之前的部分
 * @returns boolean - 是否有效
 */
function isValidLocalPart(localPart: string): boolean;

/**
 * 验证域名格式
 * @param domain - @ 之后的部分
 * @returns boolean - 是否有效
 */
function isValidDomain(domain: string): boolean;
```

## 需求映射

| Spec 需求 | Plan 实现 | 文件 | 验证方式 |
|-----------|-----------|------|----------|
| FR-001: RFC 5322 验证 | 正则表达式实现 | `src/validator.ts` | 单元测试覆盖标准邮箱格式 |

| AC-3: 错误提示 | errors.ts 错误映射 | `src/errors.ts` | 测试验证 errorCode |

## 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| RFC 5322 正则复杂度 | 中 | 中 | 使用经过验证的正则模式（参考 [RFC 5322 §3.4.1](https://tools.ietf.org/html/rfc5322#section-3.4.1) 和 [HTML5 邮箱验证正则](https://html.spec.whatwg.org/multipage/input.html#valid-e-mail-address)） |
| Unicode 邮箱支持 | 低 | 低 | v2 功能，当前仅支持 ASCII |
| 浏览器兼容性 | 低 | 中 | 使用标准 API，避免 Node.js 特有功能 |
