## 2. 功能规格

### 2.1 StoryStateManager 类

#### 2.1.1 职责
管理 Story 在 4 个阶段（create、audit、dev、post-audit）的状态转换。

#### 2.1.2 接口定义

```typescript
// 状态类型
export type StoryStage = 'create' | 'audit' | 'dev' | 'post-audit';

// 状态转换规则
export const STAGE_TRANSITIONS: Record<StoryStage, StoryStage | null> = {
  'create': 'audit',
  'audit': 'dev',
  'dev': 'post-audit',
  'post-audit': null
};

export interface StoryState {
  storyId: string;
  currentStage: StoryStage;
  stageHistory: StageEntry[];
  createdAt: Date;
  updatedAt: Date;
}

export interface StageEntry {
  stage: StoryStage;
  enteredAt: Date;
  completedAt?: Date;
  result?: 'success' | 'failed' | 'skipped';
}

// 类定义
export class StoryStateManager {
  constructor(storyId: string);

  // 获取当前状态
  getCurrentState(): StoryState;

  // 获取当前阶段
  getCurrentStage(): StoryStage;

  // 推进到下一阶段
  // 前置条件：当前阶段已完成
  // 后置条件：状态更新为新阶段
  advanceStage(): Result<StoryState, StageTransitionError>;

  // 标记当前阶段完成
  // 前置条件：存在当前阶段
  // 后置条件：当前阶段标记为完成
  completeStage(result: 'success' | 'failed' | 'skipped'): Result<StoryState, ValidationError>;

  // 检查是否可以推进到下一阶段
  canAdvance(): boolean;
}
```

#### 2.1.3 状态转换规则

```
create → audit → dev → post-audit
  ↓        ↓      ↓         ↓
初始状态  验证   开发     最终审计
```

- 不允许逆向转换
- 不允许跳过阶段
- 当前阶段必须标记为完成才能推进

### 2.2 StageValidator 类

#### 2.2.1 职责
验证每个阶段的准入条件，确保前置阶段已完成。

#### 2.2.2 接口定义

```typescript
export interface ValidationContext {
  storyId: string;
  targetStage: StoryStage;
  currentState: StoryState;
  requiredArtifacts?: string[];
}

export interface ValidationResult {
  valid: boolean;
  errors: ValidationError[];
  warnings: string[];
}

export class StageValidator {
  // 验证进入目标阶段的条件
  // 返回验证结果，包含错误列表
  validateEntry(context: ValidationContext): ValidationResult;

  // 检查特定阶段的前置条件是否满足
  private checkPrerequisites(stage: StoryStage, state: StoryState): boolean;

  // 获取阶段的前置阶段
  getPrerequisiteStage(stage: StoryStage): StoryStage | null;
}

// 错误类定义
export class ValidationError extends Error {
  code: string;
  stage: StoryStage;
  details: Record<string, unknown>;

  constructor(code: string, message: string, stage: StoryStage, details?: Record<string, unknown>);
}

export class StageTransitionError extends ValidationError {
  fromStage: StoryStage;
  toStage: StoryStage;

  constructor(fromStage: StoryStage, toStage: StoryStage, reason: string);
}
```

#### 2.2.3 阶段准入规则

| 目标阶段 | 前置阶段 | 必需完成 | 额外条件 |
|----------|----------|----------|----------|
| create | 无 | 无 | 作为初始状态 |
| audit | create | create 完成 | 无 |
| dev | audit | audit 完成 | 审计通过 |
| post-audit | dev | dev 完成 | 无 |

### 2.3 AuditLogger 类

#### 2.3.1 职责
记录每个阶段的完整生命周期，包括进入时间、完成时间和执行结果。

#### 2.3.2 接口定义

```typescript
export interface AuditLogEntry {
  id: string;
  storyId: string;
  stage: StoryStage;
  event: 'stage_entered' | 'stage_completed' | 'stage_failed';
  timestamp: Date;
  duration?: number; // 毫秒，仅 stage_completed 时存在
  input?: Record<string, unknown>;
  output?: Record<string, unknown>;
  result?: 'success' | 'failed' | 'skipped';
  executor?: string;
  metadata?: Record<string, unknown>;
}

export interface LogQuery {
  storyId?: string;
  stage?: StoryStage;
  event?: string;
  startTime?: Date;
  endTime?: Date;
}

export class AuditLogger {
  constructor();

  // 记录阶段进入事件
  logStageEntered(
    storyId: string,
    stage: StoryStage,
    input?: Record<string, unknown>,
    executor?: string
  ): AuditLogEntry;

  // 记录阶段完成事件
  logStageCompleted(
    storyId: string,
    stage: StoryStage,
    result: 'success' | 'failed' | 'skipped',
    output?: Record<string, unknown>,
    executor?: string
  ): AuditLogEntry;

  // 记录阶段失败事件
  logStageFailed(
    storyId: string,
    stage: StoryStage,
    error: Error,
    executor?: string
  ): AuditLogEntry;

  // 查询日志
  queryLogs(query: LogQuery): AuditLogEntry[];

  // 按 Story ID 查询所有日志
  getLogsByStoryId(storyId: string): AuditLogEntry[];

  // 获取日志数量
  getLogCount(): number;

  // 清空所有日志（主要用于测试）
  clearLogs(): void;
}
```

#### 2.3.3 日志存储

- 使用内存数组存储（`AuditLogEntry[]`）
- 不涉及持久化（由后续 Story 实现）
- 支持按 Story ID 查询

### 3.2 文档规范

- 所有公共类和方法必须包含 JSDoc 注释
- JSDoc 必须包含：描述、参数说明、返回值说明、可能抛出的异常
- 示例：

```typescript
/**
 * 推进 Story 到下一阶段
 *
 * @returns 成功时返回更新后的状态，失败时返回错误
 * @throws {StageTransitionError} 当不满足阶段转换条件时抛出
 *
 * @example
 * const manager = new StoryStateManager('story-001');
 * manager.completeStage('success');
 * const result = manager.advanceStage();
 * if (result.ok) {
 *   console.log('Advanced to:', result.value.currentStage);
```

### 3.3 错误处理规范

- 所有错误必须使用自定义错误类

  - `message`: 可读错误消息

### 10.1 Result 类型定义

```typescript
// 函数式结果类型，用于替代 throw
export type Result<T, E = Error> =
  | { ok: true; value: T }
  | { ok: false; error: E };

// 辅助函数
export const ok = <T>(value: T): Result<T, never> => ({ ok: true, value });
export const err = <E>(error: E): Result<never, E> => ({ ok: false, error });
```
