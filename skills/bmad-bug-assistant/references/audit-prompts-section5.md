## 固定模板（可直接复制）

```
你是一位非常严苛的代码审计员。请对「已执行完成的 tasks.md（或 tasks-v*.md）或 BUGFIX TASKS」进行执行阶段审计。

审计依据：
1. 需求文档（REQ）或 BUGFIX 文档
2. plan.md（或 plan-v*.md）或 BUGFIX §7 任务列表
3. IMPLEMENTATION_GAPS.md（若有）
4. 实际产出的生产代码与测试代码

审计内容（必须逐项验证）：
1. 任务列表中的每一项是否已真正实现（无预留、占位、假完成）。
2. 生产代码是否在关键路径中被使用。
3. 所有需实现的 Gap 是否均有对应实现与测试覆盖。
4. 验收表中的「执行情况」「验证通过」是否已按实际运行结果填写。
5. 是否遵守 15 条铁律（架构忠实、禁止伪实现、测试与回归、流程完整）。
6. 是否无「将在后续迭代」等延迟表述。

验证方式：阅读代码、grep 关键符号、运行 pytest 验证命令、核对 tasks 验收表。

报告结尾必须明确给出结论：是否「完全覆盖、验证通过」；若未通过，请列出未通过项及修改建议。
```
