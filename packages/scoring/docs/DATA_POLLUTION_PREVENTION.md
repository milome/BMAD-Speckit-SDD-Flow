# 数据污染防护

与 REQUIREMENTS §3.7、architecture §7.4、Story 4.3 一致。评测题目执行场景下，须满足以下四条防护措施。

---

## 1. 四条防护措施

| 措施 | 操作要点 | 触发条件/建议阈值 |
|------|----------|-------------------|
| **题目来源与时间隔离** | 评测题基于自有业务需求或历史项目原创设计；不采用公开题库，避免进入训练数据 | 题目入库前须标注来源；公开题库题目不得纳入题池 |
| **定期迭代题池** | 按周期更新题池，淘汰旧题，降低被训练数据覆盖风险 | 周期建议 1–2 个月；达到周期即执行题池轮换 checklist |
| **混淆变量校验** | 对题目做逻辑等价改写（变量名、表述替换）；通过率下降超过阈值时触发人工复核，不自动淘汰题目 | 阈值：受测模型在改写题上通过率下降 >20%；触发人工复核 |
| **私有闭卷与评测接口分离** | 完整题库不对外公开；仅开放评测接口或脱敏样本，避免被爬取加入训练数据 | 题库存储于私有仓库；对外仅提供评测 API 或脱敏示例 |
