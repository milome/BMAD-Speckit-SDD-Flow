## 1. 产品概述

### 1.1 背景

- **现有体系**：BMAD+Speckit 全流程各环节已有审计闭环（spec/plan/gaps/tasks/implement），部分环节有评级，但缺少全体系统一评分与能力等级，难以直接回答「模型综合能力多少分、短板在哪个环节、该轮迭代是否达标」。
- **改进目标**：在不大幅推翻现有审计闭环的前提下，增加**全体系评分评级**，并使其成为**各阶段审计闭环的重要输出与迭代结束的判定标准之一**。
- **BMAD 五层覆盖**：扩展至 BMAD 完整五层——Layer 1 产品定义（PRD/Architecture）、Layer 2 Epic/Story 规划、Layer 3 Story 开发、Layer 4 技术实现（speckit specify→plan→gaps→tasks→implement）、Layer 5 收尾（实施后审计、PR、强制人工审核）。
- **AI 代码教练与全链路 Skill**：引入 AI 代码教练角色与全链路 Code Reviewer Skill，作为「现有全流程各审计闭环的重要输出和迭代结束标准」的承载者。

### 1.2 产品定位

全体系评分评级系统，覆盖 BMAD 五层、六环节、四能力维度，作为各阶段迭代结束判定依据之一，服务于模型迭代指导、能力评估与对外能力说明等场景。

## 2. 目标用户

| 用户类型 | 使用场景 |
|----------|----------|
| 模型研发团队 | 模型迭代指导、短板定位、版本对比 |
| 工程团队 | 能力评估、对外能力说明、质量底线把控 |
| 评测运营 | 评测题目执行、横向对比、题池管理 |

---

## 3. 核心价值

1. **统一出口**：六环节分项评分 → 四能力维度聚合 → 综合百分制 → L1–L5 能力等级，作为全流程统一输出。
2. **迭代结束标准**：各阶段审计通过且（若启用评分）对应环节得分写入存储后，可视为该阶段迭代结束。
3. **可落地与可复现**：主观环节可量化、评分可复现（同交付物多评分者误差 ≤5 分）、数据污染可防、版本可追溯。

## 6. 验收标准汇总

1. **PRD 需求映射**：每个 REQ-* 需求映射到至少一个 Story。
2. **禁止词表合规**：全文无「可选」「后续」「待定」「酌情」「视情况」「先实现」「或后续扩展」。
