### 5.4 eval-question-flow E2E 是否需补充 weakness_clusters 断言？

- **现状**：plan §5.3 要求 `npm run test:scoring` 全量回归，其中包含 `scoring/__tests__/e2e/eval-question-flow.test.ts`。该 E2E 覆盖 parse→write→coach 用户可见流程，当前仅断言 `phase_scores`、`iteration_passed`。
- **分析**：实施 B06 后，coachDiagnose 将返回 `weakness_clusters`。若 eval-question-flow 使用的 fixture 数据不含足够 failed check_items（或 minFrequency≥2），`weakness_clusters` 可能为空数组，但结构仍应存在。
- **建议**：在 plan §5.2 或 §5.3 中可选补充：`eval-question-flow E2E 可补充断言 result.weakness_clusters 为数组类型（允许空数组）`，以明确用户可见流程中该字段的形态。若不补充，现有「集成测试断言 report.weakness_clusters」+「全链路回归」亦可接受，属增强建议而非必须。
