## 4. Lessons Learned

### 4.1 做得好的地方

- **契约先行**：6.3 的 query API 与 6.2/6.4 的消费约定明确；6.4 增强任务（迁移 coach-diagnose）在 spec 中预先定义
- **去重与隔离**：run_id+stage 去重、scenario=eval_question 隔离在 loader 与 index 层统一处理
- **复用而非重复**：bmad-eval-analytics Skill 不实现 discovery/coach，全部复用 coach-diagnose；scores-summary 与 coach-diagnose 共用 query 层

### 4.2 挑战与改进点

- **6.5 progress 标注**：批判审计员要求「T2.2 自然语言触发验收」在 progress 中显式说明；子代理易将 progress 作完成清单而非验收证据，需在 prompt 中强调
- **GAP-024 未实现**：组合 queryByFilters API 仍归属 Deferred，Epic 7 若需复杂筛选可能需临时多次 query+filter
