# BUGFIX：可解析评分块禁止用描述代替结构化块

**日期**：2026-03-06  
**发现场景**：审计报告输出「可解析评分块（总体评级 X，维度分 Y–Z）」等文字描述，导致 parseAndWriteScore 解析失败或 dimension_scores 为空，仪表盘不显示四维雷达图  
**文档路径**：`_bmad-output/implementation-artifacts/_orphan/BUGFIX_可解析评分块禁止描述代替结构化块.md`

---

## §1 现象/问题描述

### 1.1 现象

审计报告（spec/plan/gaps/tasks 等阶段）中，code-reviewer 有时输出**对可解析块的文字描述**（如「可解析评分块（总体评级 A，维度分 92–95）」），而非 `parseAndWriteScore` 所需的**结构化块**，导致：

- 解析器 `extractOverallGrade` 可能因格式不符（如 `总体评级: A-`、`C+`）无法匹配
- 解析器 `parseDimensionScores` 无法从「维度分 92–95」等描述中提取四维分数，返回空数组
- 仪表盘 `dimension_scores` 为空，四维雷达图显示「无数据」
- 即便 phase_score 可计算，仪表盘展示不完整

### 1.2 复现/发现场景

- 审计结论或总结中写：「可解析评分块（总体评级 A，维度分 92–95）」「包含可解析评分块（总体评级 C）」
- 报告中仅有一行 `总体评级: C`，无 `维度评分:` 及四行 `- 维度名: XX/100`
- 使用 `总体评级: A-`、`C+` 等非标准等级
- 将「维度分」写作区间或概括句（如「各维度 92–95」），非逐行格式

### 1.3 预期行为

按 audit-prompts §4.1、audit-prompts-critical-auditor-appendix §7、`scoring/parsers/audit-generic.ts`、`scoring/parsers/dimension-parser.ts` 约定：

- 审计报告**必须**在结尾输出**完整的结构化块**，包含：
  - 独立一行 `总体评级: [A|B|C|D]`（仅限 A/B/C/D，不得使用 A-、C+ 等）
  - 独立段落 `维度评分:` 及四行 `- 维度名: XX/100`（需求完整性、可测试性、一致性、可追溯性）
- 不得用「可解析评分块（总体评级 X，维度分 Y–Z）」等文字**描述**代替上述**结构化输出**
- 结构化块须为解析器可识别的精确格式，否则 parseAndWriteScore 无法正确解析，仪表盘无法显示四维数据

### 1.4 技术依据

| 解析器 | 正则/要求 | 无效输入示例 |
|--------|-----------|--------------|
| extractOverallGrade | `/总体评级:\s*([ABCD])/` | `总体评级: A-`、`C+`、内嵌于描述句 |
| parseDimensionScores | `维度名: XX/100`（行级） | `维度分 92–95`、`各维度 90+`、区间表述 |

## §2 根因分析

### 2.1 根因结论

**根因**：audit-prompts 与 critical-auditor-appendix 未明确禁止「用描述代替结构化块」，code-reviewer 可能将「附可解析评分块」理解为在总结或正文中**提及**块内容，而非输出**完整的、可被正则解析的 markdown 块**。

1. **约束缺失**：未在 prompt 或附录中写明「禁止用描述代替」「必须输出完整结构化块」。
2. **示例不足**：虽有示例，但未强调「描述句≠结构化块」、未列出反例（如「可解析评分块（总体评级 A，维度分 92–95）」为无效）。
3. **等级约束未写**：未明确总体评级仅限 A/B/C/D，导致出现 A-、C+ 等，正则不匹配。
4. **summary 与 body 混淆**：子代理在 summary 中写「包含可解析评分块（…）」时，可能未在报告正文中输出实际块，或仅在 summary 中描述。

### 2.2 根因链条

| 层级 | 问题 | 后果 |
|------|------|------|
| audit-prompts §4.1 | 要求「包含可解析块」但未禁止描述替代 | 模型输出描述句而非结构化块 |
| critical-auditor-appendix §7 | 有示例格式，缺「禁止描述」约束与反例 | 模型可误将概括视为合规 |
| 总体评级 | 未限定 A/B/C/D，未禁止 A-、C+ | 正则不匹配，extractOverallGrade 返回 null |
| 维度分 | 未禁止区间/概括（如 92–95、各维度 90+） | parseDimensionScores 返回空数组 |

### 2.3 证据与引用

- **scoring/parsers/audit-generic.ts 第 22–24 行**：`extractOverallGrade` 仅匹配 `总体评级:\s*([ABCD])`。
- **scoring/parsers/dimension-parser.ts 第 13 行**：`DIMENSION_SCORE_PATTERN` 要求 `维度名: 数字/100` 行级格式。
- **用户反馈**：「总体评级没有展开可解析评分块」「这样显示的也有，也是不全」——描述式输出导致解析/展示不全。

## §4 修复方案

### 4.1 原则

- **禁止用描述代替结构化块**：不得用「可解析评分块（总体评级 X，维度分 Y–Z）」等文字概括；必须在报告中输出**完整的结构化块**。
- **总体评级仅限 A/B/C/D**：不得使用 A-、C+ 等变体。
- **维度评分须逐行**：四维必须各占一行，格式为 `- 维度名: XX/100`。

### 4.3 新增约束条文（建议原文）

**禁止用描述代替结构化块**：不得在总结或正文中用「可解析评分块（总体评级 X，维度分 Y–Z）」等文字概括；必须在报告中输出**完整的结构化块**，包含：独立一行 `总体评级: X` 以及四行 `- 维度名: XX/100`。总体评级只能是 **A/B/C/D**（不得使用 A-、C+ 等）。维度分须逐行写出，不得用区间或概括（如「92–95」「各维度 90+」）代替。

### 4.4 反例（无效输出）

| 无效写法 | 问题 |
|----------|------|
| `可解析评分块（总体评级 A，维度分 92–95）` | 描述句，非结构化块 |
| `总体评级: A-` / `C+` | 非 A/B/C/D，正则不匹配 |
| `维度分 92–95` | 区间、缺具体维度名、格式不符 |
| 仅 `维度评分:` 标题无下文 | 无 `- 维度名: XX/100` 行，解析不到 |

### 4.5 完整示例（正确输出）

```markdown
## 可解析评分块（供 parseAndWriteScore）

总体评级: A

维度评分:
- 需求完整性: 95/100
- 可测试性: 92/100
- 一致性: 94/100
- 可追溯性: 93/100
```

## §5 流程/建议流程（可选）

### 5.1 code-reviewer 输出自检

审计报告生成后，输出前自检：
1. 是否包含独立段落 `## 可解析评分块（供 parseAndWriteScore）`？
2. 该段落是否含 `总体评级: [A|B|C|D]`（仅 A/B/C/D）？
3. 是否含 `维度评分:` 及四行 `- 需求完整性: XX/100` 等？
4. 是否避免了「可解析评分块（…）」式描述句替代实际块？

### 5.2 主 Agent 或格式校验脚本

若 report 中可解析块部分匹配到「可解析评分块（总体评级」「维度分 Y–Z」等描述模式，且无完整四行 `- 维度名: XX/100`，可判定格式不合格，要求重新生成。
