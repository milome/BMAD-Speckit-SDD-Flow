| 验收项 | 命令形式 | 独立可执行 | 预期（实施后） |
|--------|----------|------------|----------------|

#### 3. 锚点是否可能出现在「审计报告正文」中造成混淆？

| 检查点 | 分析 |
|--------|------|
| **锚点位置** | 锚点插入在 audit-prompts 的 **prompt 模板**内（供 code-reviewer 参考的指令），非报告解析器预期格式 |
| **报告输出来源** | 审计报告由 code-reviewer 根据 prompt 生成；prompt 要求「报告结尾必须包含 §4.1 规定的可解析评分块」 |
| **锚点用途** | 锚点为 TASKS 验收而设的元标记，用于 grep 计数，非用户可见内容的设计目标 |
| **泄漏风险** | 若 agent 误将锚点复制进报告，报告会含【§1 可解析块要求】等。parseAndWriteScore 解析的是 §4.1 定义的「可解析评分块」格式（总体评级、维度评分），非锚点；锚点若出现于报告，不会匹配解析规则，**不造成解析失败** |
| **用户混淆** | 若锚点出现在报告正文，用户可能看到无意义标记，造成轻微困惑；属低概率、低影响 |
| **建议** | T12 注中可补充「锚点仅用于本任务验收，不要求出现在审计报告正文」；**非阻塞** |

**判定**：锚点应在 prompt 模板内，供 code-reviewer 参考；不应出现在用户可见的审计报告输出中。当前设计满足此要求；泄漏风险低，不构成本轮 gap。
