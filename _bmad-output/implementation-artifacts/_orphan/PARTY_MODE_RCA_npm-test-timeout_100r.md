## 最终方案（共识）

1. **根因**：两用例在默认 5000ms 下超时，主因为 I/O 与进程冷启动；未排除潜在逻辑问题，但延长 timeout 可先验证环境假设。
2. **方案**：采用方案 A（为两用例单独设置 `{ timeout: 15000 }`），不改动 vitest 全局配置。
3. **Fallback**：若 `npm test` 仍超时，将 15000 调整为 20000 并重跑；若仍失败，调整为 30000。再失败则进入方案 C（优化用例）或 B（排除），需单独决策。
