## 一、关切点1: BMAD PRD需求环节及架构文档环节缺失

### 1.1 问题分析

原方案从Create Story开始，跳过了BMAD前期的关键环节：
- **Product Brief**: 产品概述、目标用户、核心价值
- **PRD**: 详细需求、验收标准、优先级
- **Architecture**: 技术架构、模块划分、接口定义
- **Epic/Story定义**: 功能拆分、依赖关系

**缺失后果**:
1. Story验收标准可能与原始业务需求脱节
2. speckit的技术实现可能偏离架构约束
3. 需求变更无法追溯到下游

### 1.2 解决方案

#### 1.2.1 流程架构修订

```
修订后的BMAD-Speckit整合架构：

Layer 1: 产品定义（新增）
  └─ Product Brief → PRD → Architecture
  └─ 产出：产品概述、需求文档、架构设计

Layer 2: Epic/Story规划（新增）
  └─ create-epics-and-stories
  └─ 产出：Epic列表、Story列表（粗粒度）、依赖关系

Layer 3: Story开发（现有）
  └─ Create Story（细化）→ party-mode → Story文档（细粒度）→ 审计
  └─ PRD需求追溯、Architecture约束传递

Layer 4: 技术实现（现有）
  └─ specify → plan → GAPS → tasks → TDD执行
  └─ 需求映射（PRD→Story→spec→task）

Layer 5: 收尾（现有）
  └─ 实施后审计、Epic集成、发布
```

#### 1.2.2 需求追溯链扩展

原映射表:
| 需求ID | 需求描述 | 对应Story | 对应spec章节 | 对应task | 状态 |

扩展为:
| PRD需求ID | PRD需求描述 | Architecture组件 | Story | spec章节 | task | 状态 |

#### 1.2.3 需求变更管理

新增机制:
1. **版本追踪**: PRD/Architecture文档版本化
2. **变更检测**: Create Story时检查版本差异
3. **影响评估**: 标记受影响的Story状态
4. **同步机制**: 更新Story文档、spec.md、tasks.md

## 二、关切点2: Epic级Worktree策略优化

### 2.1 问题分析

原方案根据Story数量和文件重叠度决定worktree级别，策略复杂：
- Story数 ≤ 2: Story级worktree
- Story数 3-8，文件重叠低: Epic级worktree
- Story数 3-8，文件重叠高: Story级worktree
- Story数 > 8: 建议拆分Epic

**用户诉求**: 一个Epic最好只有一个worktree

**潜在问题**:
1. Story间文件冲突如何处理？
2. 并行开发需求如何满足？
3. 中间Story回退如何处理？

### 2.2 解决方案

#### 2.2.1 简化策略

```
### Worktree核心策略（修订版）

核心原则:
- 一个Epic默认只创建一个worktree
- Story在Epic worktree内以分支形式管理
- 支持串行/并行模式切换
```

#### 2.2.2 执行模式

**串行模式（默认）**:
```
Story 4.1 → merge到feature-epic-4
    ↓
Story 4.2（基于merge后的feature-epic-4）→ merge
    ↓
Story 4.3 → ...
```

**并行模式（可选）**:
```
Story 4.1 ─┐
           ├─→ 并行开发 → 逐个merge + 冲突解决审计
Story 4.2 ─┘
```

#### 2.2.3 文件范围预测

- Create Story时预测文件修改范围（基于Architecture模块文件映射）
- 并行Story文件范围有重叠时警告
- **注意**: 预测仅供参考，用户可手动调整

#### 2.2.4 冲突处理

1. **冲突检测**: 并行Story merge时检测文件冲突
2. **冲突解决**: 用户手动解决冲突
3. **冲突审计**: 触发code-review审计，验证两个Story的测试通过

#### 2.2.5 模式切换

```bash
# 从串行切换到并行
/bmad-set-worktree-mode epic=4 mode=parallel

# 从并行切换到串行
/bmad-set-worktree-mode epic=4 mode=serial

# 回退到Story级
/bmad-set-worktree-mode epic=4 mode=story-level
```
