```markdown
### 1.0 spec 目录路径约定（BMAD 与 standalone 双轨制）

**BMAD 流程**（当 speckit 由 bmad-story-assistant 嵌套触发时）：
- 路径格式：`specs/epic-{epic}/story-{story}-{slug}/`
- **slug 必选**，保证目录可读性；若省略则导致 `specs/epic-4/story-1/` 等纯数字命名，可读性差。
- 示例：`specs/epic-4/story-1-implement-base-cache/`
- 产出文件名：`spec-E{epic}-S{story}.md`、`plan-E{epic}-S{story}.md`、`tasks-E{epic}-S{story}.md`、`IMPLEMENTATION_GAPS-E{epic}-S{story}.md`

**slug 来源规则**（按优先级，若无法推导则要求用户显式提供）：
| 优先级 | 来源 | 示例 |
|--------|------|------|
| 1 | Story 文档标题（取前若干词，转 kebab-case） | "Implement base cache class" → `implement-base-cache` |
| 2 | Epic 名称（去掉 `feature-` 前缀） | `feature-metrics-cache` → `metrics-cache` |
| 3 | Story scope 首句关键词（转 kebab-case） | "缓存服务基础实现" → `cache-service-base` |
| 4 | 兜底 | `E4-S1` 作为 slug（保证唯一，可读性最差） |

**standalone 流程**（用户直接执行 speckit，未走 BMAD）：
- 路径格式：`specs/{index}-{feature-name}/`
- index 由 create-new-feature.ps1 的 Get-HighestNumberFromSpecs 推导
- 示例：`specs/015-indicator-system-refactor/`

**fallback 规则**：无 `--mode bmad` 或 `--epic`/`--story` 参数时，使用 standalone 行为；有 `--mode bmad` 时，必须提供 `--slug` 或从 Story 文档推导。
```

## T-SPEC-2：create-new-feature.ps1 支持 BMAD 模式

### 修改路径

`{project-root}/_bmad/scripts/bmad-speckit/powershell/create-new-feature.ps1`

### 具体修改内容

**修改 1**：在 `param()` 块中新增参数（在 `[Parameter(ValueFromRemainingArguments = $true)]` 之前插入）：

```powershell
[switch]$ModeBmad,
[int]$Epic = 0,
[int]$Story = 0,
[string]$Slug = ""
```

**修改 2**：在 `$specsDir = Join-Path $repoRoot 'specs'` 及 `New-Item -ItemType Directory -Path $specsDir -Force` 之后（约第 197 行）、`Get-BranchName` 函数定义之前，新增 BMAD 模式分支逻辑：

```powershell
# BMAD mode: create specs/epic-{epic}/story-{story}-{slug}/
if ($ModeBmad) {
    if ($Epic -le 0 -or $Story -le 0) {
        Write-Error "BMAD mode requires -Epic N -Story N (N>=1)"
        exit 1
    }
    if ([string]::IsNullOrWhiteSpace($Slug)) {
        $Slug = "E$Epic-S$Story"
    }
    $targetDir = Join-Path $specsDir "epic-$Epic"
    $storyDirName = "story-$Story-$Slug"
    $fullPath = Join-Path $targetDir $storyDirName
    $branchName = "story-$Epic-$Story"
    $specFile = Join-Path $fullPath "spec-E$Epic-S$Story.md"
    if (-not (Test-Path $fullPath)) {
        New-Item -ItemType Directory -Path $fullPath -Force | Out-Null
        @("# Spec E$Epic-S$Story`n`n*Generated by create-new-feature.ps1 -ModeBmad*") | Set-Content -Path $specFile -Encoding UTF8
    }
    if ($Json) {
        [PSCustomObject]@{ BRANCH_NAME = $branchName; SPEC_FILE = $specFile; SPEC_DIR = $fullPath } | ConvertTo-Json -Compress
    } else {
        Write-Output "BRANCH_NAME: $branchName"
        Write-Output "SPEC_FILE: $specFile"
        Write-Output "SPEC_DIR: $fullPath"
    }
    exit 0
}
```

**修改 3**：在 Help 文本中新增 BMAD 模式说明（在 Examples 之后）：

```powershell
Write-Host "  -ModeBmad          BMAD mode: create specs/epic-N/story-N-{slug}/"
Write-Host "  -Epic N            Epic number (required when -ModeBmad)"
Write-Host "  -Story N           Story number (required when -ModeBmad)"
Write-Host "  -Slug <name>       Slug for directory name (required or derived from Story doc)"
Write-Host ""
Write-Host "BMAD example:"
Write-Host "  ./create-new-feature.ps1 -ModeBmad -Epic 4 -Story 1 -Slug implement-base-cache"
```

### 验收标准

- [ ] 支持 `-ModeBmad -Epic 4 -Story 1 -Slug implement-base-cache` 创建 `specs/epic-4/story-1-implement-base-cache/`
- [ ] 无 `-Slug` 时使用 `E4-S1` 作为兜底
- [ ] 无 `-ModeBmad` 时保持原有 015-feature-name 行为不变
- [ ] Help 文本包含 BMAD 参数说明
