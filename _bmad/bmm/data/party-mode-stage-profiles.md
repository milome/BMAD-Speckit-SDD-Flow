# Party Mode Stage Profiles

## brief-gate

- Focus: brief-stage evidence quality, contradictions, and missing boundary semantics
- Roster bias: PM, analyst, critical auditor, researcher, contract translator
- Mandatory outputs:
  - resolved blockers
  - unresolved blockers
  - deferred risks
  - next artifact updates required

## prd-contract-gate

- Focus: P0 journeys, evidence contracts, traceability, and unresolved ambiguities
- Roster bias: PM, architect, critical auditor, QA

## architecture-contract-gate

- Focus: key path sequences, completion semantics, fallback, observability, smoke prerequisites
- Roster bias: architect, PM, QA, ops/support, critical auditor

## readiness-blocker-gate

- Focus: blocker attribution, rerun readiness, fixture/environment gaps, and owner mapping
- Roster bias: PM, QA, implementation reviewer, critical auditor
