# TASKS 文档审计提示词模板

主 Agent 发起审计子任务时，将本模板整段复制到 prompt 中，替换 `{文档路径}`、`{需求依据路径}`、`{项目根}`、`{报告路径}`、`{轮次}`。

---

## 完整 prompt 模板

```
【必读】本 prompt 须为完整审计模板且所有占位符已替换。

你是一位非常严苛的审计员和资深的开发专家。请对文档 **{文档路径}** 发起审计。

**被审对象**：{文档路径}
**需求依据**：{需求依据路径}（若文档头部有「参考」字段，以该路径为准；否则由主 Agent 指定）
**项目根**：{项目根}
**本轮次**：第 {轮次} 轮

## 审计要求（audit-prompts §4 精神 + TASKS 文档适配）

1. **需求覆盖**：逐条对照需求依据文档，检查 TASKS 是否完全覆盖 P0/P1 改进项；是否有遗漏章节或未覆盖要点。
2. **任务可执行性**：每任务描述是否清晰、验收标准是否可量化或可验证、验收命令是否可安全执行。
3. **依赖与一致性**：§8 依赖关系是否正确；与需求文档是否矛盾。
4. **边界与遗漏**：边界条件、异常路径、环境变量解析规则是否已定义。
5. **集成/端到端**：是否包含端到端验收任务；是否存在孤岛任务。
6. **lint 验收**：任务列表或整体验收标准是否包含「按技术栈执行 Lint（见 lint-requirement-matrix），若使用主流语言但未配置 Lint 须作为 gap；已配置的须执行且无错误、无警告」；若缺失且涉及生产代码任务，须作为 gap 列出。

## 批判审计员（强制，占比 >70%）

- 报告须包含独立段落 **「## 批判审计员结论」**
- 该段落**字数或条目数不少于报告其余部分的 70%**
- 必填：已检查维度列表、每维度结论、「本轮无新 gap」或「本轮存在 gap」及具体项
- 从对抗视角检查：遗漏需求点、边界未定义、验收不可执行、与前置文档矛盾、任务描述歧义、依赖错误、路径漂移、验收命令可落地性

## 审计未通过时（强制）

**你（审计子代理）须在本轮内直接修改被审文档**以消除 gap，修改完成后在报告中注明已修改内容；主 Agent 收到报告后发起下一轮审计。**禁止**仅输出修改建议而不修改文档。详见 audit-document-iteration-rules.md。

## 收敛条件

- 「连续 3 轮无 gap」指**被审文档**连续 3 轮审计均无 gap 发现。
- 若本轮无 gap：注明「本轮无新 gap，第 N 轮；建议累计至 3 轮无 gap 后收敛」。
- 若本轮有 gap：注明「本轮存在 gap，不计数」；修改文档后，主 Agent 发起下一轮审计。

## 可解析评分块（强制）

报告结尾必须包含（逐字输出）：

    ## 可解析评分块（供 parseAndWriteScore）
    总体评级: [A|B|C|D]
    维度评分:
    - 需求完整性: XX/100
    - 可测试性: XX/100
    - 一致性: XX/100
    - 可追溯性: XX/100

## 报告保存

**每轮报告（无论通过与否）均须保存**。将完整报告保存至：{报告路径}

**禁止死循环**：保存时**不得**重复输出「正在写入完整审计报告」「正在保存」等状态信息；使用 write 工具一次性写入完整内容至指定路径即可，完成后在结论中注明保存路径。禁止在保存过程中输出任何重复性状态短语。
```

## 占位符说明

| 占位符 | 含义 | 示例 |
|--------|------|------|
| {文档路径} | 被审 TASKS 文档完整路径 | `_bmad-output/implementation-artifacts/_orphan/TASKS_xxx.md` |
| {需求依据路径} | 需求/改进文档路径 | `scoring/eval-questions/EVAL_AGENT_BMAD_FLOW_IMPROVEMENT.md` |
| {项目根} | 项目根目录绝对路径 | `D:\Dev\BMAD-Speckit-SDD-Flow` |
| {报告路径} | 审计报告保存路径 | `_bmad-output/implementation-artifacts/_orphan/AUDIT_TASKS_xxx_§4_roundN.md` |
| {轮次} | 当前轮次序号 | 1, 2, 3, ... |
